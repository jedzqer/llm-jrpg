# LLM JRPG

`LLM JRPG` 是一个用于开发基于大语言模型的角色扮演游戏的起始项目。
当前技术栈刻意保持收敛，先保证原型能快速跑起来：

- 前端：`Next.js` + `React` + `AI SDK UI`
- 后端：`Node.js` 路由处理 + `AI SDK Core`
- 游戏领域层：本地 TypeScript 模块，负责玩家状态、NPC 状态、任务状态和世界上下文

这个仓库的目标不是一开始就堆复杂框架，而是先把“可玩的叙事循环”搭出来，再逐步扩展成完整 JRPG 系统。

**本项目旨在探索 LLM 文字角色扮演游戏与常规游戏系统的结合**

## AI 创作剧情流动

当前项目已经实现“先创作剧情，再落地结构化游戏状态”的双阶段流程，用来平衡叙事自由度与系统可控性。

1. 前台对话阶段：AI 先专注生成剧情、场景和角色互动，不直接暴露工具信息，避免叙事过程被工具调用打断。
2. 后台解析阶段：单次剧情输出结束后，系统会再次调用 AI，对本轮内容进行结构化解析，判断是否需要创建或更新 NPC 实体。
3. 实体落地阶段：如果剧情中引入了新的 NPC，AI 会通过 tool call 创建对应实体，并补全其状态、物品、技能等战斗与交互所需信息；如果不需要更新，则允许跳过。
4. 战斗接管阶段：当剧情进入遭遇战时，AI 通过 tool call 调用战斗系统，由游戏战斗系统正式托管战斗流程，并读取 NPC 已生成的技能与道具参与结算。
5. 信息补全阶段：如果战斗或剧情推进过程中发现 NPC 数据不完整，系统会再次调用 AI 补全缺失信息，保证战斗与后续叙事可继续执行。

为保证状态一致性，后台工具调用期间会暂时锁定用户输入，防止玩家在状态尚未写入完成时继续推进对话。

此外，项目还将对“长期未登场后再次出现”的 NPC 进行重新生成或刷新，避免角色信息长期停滞，与当前剧情阶段脱节。

## 当前状态

项目基础框架已经完成，`/api/chat` 已接入真实的流式 LLM 调用，并兼容 OpenAI-compatible API。
现在对话会话、消息历史、世界状态、手动存档和 LLM 配置都持久化到 `SQLite`，模型参数直接在应用界面里配置，不再依赖外部配置文件。

玩家可以随时手动存档；一旦战斗失败死亡，输入会被锁定，只能通过读档回到最近一次存档。此前的“删除消息”功能已移除，避免消息历史与世界状态回滚不一致。

## 快速开始

1. 安装依赖：

```bash
npm install
```

2. 启动开发服务器：

```bash
npm run dev
```

3. 打开 `http://localhost:3000`

4. 在页面顶部的“模型配置”面板中填写：

- `Provider 名称`：默认 `openai`
- `模型 ID`：例如 `gpt-4o-mini`
- `Base URL`：兼容服务时填写，官方 OpenAI 可留空
- `API Key`：首次保存必填

5. 保存后即可开始对话，配置会写入 `data/app.db`

6. 点击“新建游戏”后，先完成建角：

- 自定义角色姓名
- 自定义出身 / 门派
- 选择灵根
- 在初始总点数内分配 `气血` 与 `灵力`

完成后才会正式创建新会话并进入游戏，不再直接使用默认角色与默认数值。
